Miioo API 前后端差异报告

生成时间：2026-05-28  |  对照文档：api文档.json（OpenAPI 3.1）

✅ 修复状态：所有 路径错误 和 字段差异 已全部修复，vite build 构建通过。 页面通过 legacy alias 兼容旧调用，无需修改页面代码即可运行。 前端缺失 和 后端无接口 项暂以 mock / no-op 保留，待后续与后端沟通后补充。
说明：以下表格覆盖 src/api/ 全部 8 个文件。 路径错误表示前端调用路径与后端不符， 字段差异表示请求体字段名/结构与后端 schema 不一致， 前端缺失表示后端有此接口但前端未实现， 后端无接口表示前端有此函数但后端无对应路由。 已修复表示已在 API 层修正，页面代码无需改动。

六、修复优先级建议

优先级问题类型影响
P0路径错误（表一）直接 404，功能完全不可用
P1请求体字段差异（表二）422 验证错误，接口调用失败
P2前端缺失的必要接口（表三中标记为页面用到的）页面功能无数据支撑
P3后端响应字段未消费（表四）功能可跑通但数据展示不完整
P4后端无接口的函数（表五）需与后端沟通规划，暂可保留 mock
