Miioo API 前后端差异报告

更新时间：2026-06-04  |  对照文档：api文档latest.json（OpenAPI 3.1） |  上次生成：2026-06-03

✅ 本轮更新（2026-06-04）：对照 api文档latest.json 与上版差异，新增 10 个接口（手机/微信绑定、卡片可见性、Fal 配置、项目模板、主体提取）， 2 处响应字段新增（display_id/phone_bound/wechat/wechat_bound）。 微信绑定/手机换绑后端接口已落地，从"后端无接口"迁移至"前端缺失"。 新发现 标记为本次新增条目。
说明： 路径错误 前端调用路径与后端不符   字段差异 请求体字段名/结构不一致   前端缺失 后端有路由前端无函数   后端无接口 前端有函数后端无路由   已修复 本轮已对齐   新发现 上轮报告未覆盖

一、路径错误（当前仍存在）

上轮全部路径问题已修复。以下为本轮新发现的路径差异。

文件 / 函数前端当前路径后端正确路径差异说明
settings.js
apiGetWatermarkSettings
apiUpdateWatermarkSettings		 GET/POST /api/settings/watermark （后端无此路由） 路径错误新发现后端无 /api/settings/* 路由。水印开关存于 ProviderResponse 的 default_image_watermark / default_video_watermark 字段，应通过 PATCH /api/providers/{id} 更新
config.js
apiGetBanner		 GET /api/config/banner GET /api/api-config/banner 路径错误新发现前缀 /api/config 应为 /api/api-config
user.js

auth.js
apiLogout		 仅本地清 token，未调后端 POST /api/auth/logout 前端缺失新发现后端有 logout 接口，当前前端只清本地 token，服务端 session 未失效
creation.js
音频任务轮询（缺函数）		 （无独立函数，复用图片 tasks） GET /api/creation/audios/tasks/{task_id} 前端缺失新发现视频有 apiGetCreationVideoTask，音频缺少对应的 apiGetCreationAudioTask，应用 /api/creation/audios/tasks/{task_id}
