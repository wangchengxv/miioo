# miioo 项目上下文

## 产品定位
AIGC 影视化工作流产品，面向影视创作者。当前阶段：业务页面开发中。

## 技术栈
- React 19 + Tailwind CSS v4 + Vite 8
- 仅深色主题，无浅色切换
- Token 通过 `@theme` 注册在 `src/index.css`

## 开发核心规则

### 优先级（从高到低）
1. 设计稿代码 → 完全按设计稿复刻，不自行发挥
2. Token 替换 → 颜色/圆角/字号必须替换为 token 类名
3. 组件文档规范 → `design-system/components/` 下对应文档
4. frontend-design 插件 → 无设计稿代码时，用于制定视觉与间距规则

### 样式规则
- 颜色必须用 Token 类名，禁止硬编码（无对应 token 时除外）
- 圆角用 `rounded-medium`，字号用 `text-font-size-14` 等 token
- 有设计稿代码时，直接复刻设计稿代码里的间距数值

### 常用 Token 对照
```
bg-[#161616]   → bg-neutral-200
bg-[#111111]   → bg-neutral-400（页面底色）
bg-[#090909]   → bg-neutral-500（工具栏）
bg-[#00000033] → bg-black-20
bg-[#FFFFFF1A] → bg-white-10
text-white     → 保留
```

### 间距规则
- 有设计稿代码时，直接复刻设计稿代码里的间距数值
- 没有设计稿代码时，必须先调用 frontend-design 插件制定间距规则，拿到具体数值后再开始写代码，不可自行估算间距

### 文件结构
```
src/components/   通用组件
src/layouts/      页面框架（侧边栏、顶栏）
src/pages/        业务页面
```
- 组件文件名大驼峰：`ProjectCard.jsx`
- 每个组件只做一件事，复杂组件拆分子组件

### 开发前必读
- `design-system/tokens.md` — 所有可用 token
- 对应组件文档 `design-system/components/xxx.md`

## 开发流程（四步法）

每个页面或功能模块按以下四步推进，**每步完成并验收后再进入下一步**，不可跳步。

### Step 1 — 布局骨架
只做区域划分，不填充组件，不处理样式细节。

**指令模板：**
```
请根据以下描述生成页面布局骨架。
只需要区域划分，不需要填充任何组件和样式细节。
背景色使用 bg-surface-content-area。

[在此描述页面区域划分]
```

**验收标准：**
- 区域比例和位置正确
- 无具体组件和样式细节

---

### Step 2 — 组件填充
在骨架基础上填充组件，必须遵循设计规范。

**指令模板：**
```
请在上一步的布局基础上填充组件。
必读：design-system/tokens.md、design-system/components/[相关组件文档]。
禁止使用硬编码色值，所有样式必须使用 token 类名。

[在此描述每个区域放哪些组件]
```

**验收标准：**
- 组件类型和位置正确
- 无硬编码色值，token 类名使用规范

---

### Step 3 — 交互逻辑
单独处理状态和行为，不修改已有布局和样式。

**指令模板：**
```
请在上一步的基础上添加交互逻辑，不要修改已有的布局和样式。

[在此描述交互行为和状态处理]
```

**验收标准：**
- 交互行为符合预期
- 边界状态（loading、error、disabled）都有处理

---

### Step 4 — 细节修正
针对具体问题小幅修改，不重新生成。

**指令模板：**
```
请只修改以下问题，不要改动其他部分：

[在此列出具体问题]
```

**验收标准：**
- 问题修复，其他部分未受影响
