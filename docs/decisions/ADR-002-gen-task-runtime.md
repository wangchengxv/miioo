# ADR-002：生成型能力统一收口为任务制运行时

## 状态

已批准（2026-06-10）

## 背景

主体提取、智能分镜、主体图生成、视频生成、剪辑导出等能力具有明显的长耗时特征。早期同步整包返回模式会导致超时、页面假死和错误难定位。

## 可选方案

### 方案 A：继续同步返回

优点：

- 接口表面更简单

缺点：

- 易超时
- 无法承接部分完成
- 页面难以实时刷新中间结果

### 方案 B：统一任务制 + 轮询

优点：

- 页面可先拿到 `taskId`
- 可承接进行中、成功、失败、部分完成
- 便于后端扩展子项级进度

缺点：

- 需要更完整的任务状态模型
- 页面需要统一轮询逻辑

### 方案 C：统一 SSE

优点：

- 体验更实时

缺点：

- 当前联调复杂度更高
- 服务端和前端都要额外维护连接状态

## 最终决策

当前阶段采用方案 B，统一优先使用 `GenTask + task items + /api/tasks/{task_id}`。SSE 作为后续增强而非首要闭环。

## 落地后果

- 新增长耗时生成接口时，优先设计成任务制
- 前端必须把 `partial` 当作有效终态之一承接
- 文档必须记录任务状态语义和页面回显规则
