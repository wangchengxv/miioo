# ADR-003：Harness 文档改为入口页 + 小文档 + 历史归档

## 状态

已批准（2026-06-10）

## 背景

项目已积累大量联调成果，但文档逐步膨胀。`PROJECT.md`、全局进度文档、`HARNESS_P0_HISTORY.md` 等文件包含了过多历史细节，导致当前信息查找成本过高。

## 可选方案

### 方案 A：继续沿用大文件

优点：

- 历史连续，改动少

缺点：

- 读取成本持续上升
- 当前事实和历史事实混杂
- AI 上下文容易被历史噪声占满

### 方案 B：直接删除旧文档并重写

优点：

- 结构最干净

缺点：

- 历史沉淀容易丢失
- 对现有协作入口破坏过大

### 方案 C：保留旧文档为归档，新建分层小文档承接当前事实

优点：

- 历史保留
- 当前入口更清晰
- 后续维护成本更低

缺点：

- 需要定义清楚各文档职责边界

## 最终决策

采用方案 C。保留现有大文件作为历史母本或专项归档，同时新增 `docs/` 分层文档、`CHANGELOG.md` 与阶段进度子文档。

## 落地后果

- 新增进展优先进入小文档
- 历史大文件不再作为默认阅读入口
- 根级导航文档必须指向新的小文档结构
