# Backlog 索引

## 1. 定位

本索引用于把阶段优先级和 backlog 从单个文件拆成按阶段阅读的小文档。

根级 `HARNESS_P0_BACKLOG.md` 继续保留为固定入口，但默认阅读顺序切到这里。

## 2. 阅读顺序

1. 当前活跃项：`backlog-active-runtime.md`
2. 后置与保留项：`backlog-deferred-followups.md`
3. 主题专项：`backlog-media-pipeline-bplus.md`
4. 历史母本：`backlog-legacy-2026-06-10.md`

## 3. 使用原则

- 先看当前活跃项，再决定是否需要进入页面级 Todo
- 若任务属于“生成资产企业级媒体交付升级”，优先转到 `backlog-media-pipeline-bplus.md`
- 若需要一页看到 `MEDIA-BPLUS` 的主方案、实施清单、backlog、专项设计稿与 runbook 模板入口，优先转到 `media-bplus-execution-index.md`
- 若任务涉及页面闭环，再配合 `page-mapping-index.md` 一起阅读
- 完成任务后，优先回写对应子文档，再同步 `module-progress.md` 与 `CHANGELOG.md`
