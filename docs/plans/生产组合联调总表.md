# 生产组合联调总表

## 1. 文档定位

本文档用于把“原系统事实基线 + 新生产前端 + 当前生产后端”的组合关系按模块拆成可执行联调总表。

统一口径：

- 原系统事实基线：`docs/`
- 生产前端：`frontend/`
- 生产后端：`backend/`
- 当前线上后端域名：`https://www.miiooai.com`

默认使用方式：

1. 先看对应模块表，确认原系统事实与当前代码现实
2. 再按“差异断点 -> 必须改动位置 -> 上线前配置项”执行
3. 完成后回写 `项目进度文档.md`、`module-progress.md`、`CHANGELOG.md`

## 2. 总体结论

| 维度 | 当前判断 |
|---|---|
| 主组合口径 | 已明确，`backend` 为生产后端、`frontend` 为生产前端、`docs` 为原系统事实基线 |
| 当前部署锚点 | 前端生产 `VITE_API_BASE_URL` 与后端 `PUBLIC_BASE_URL` 均应指向 `https://www.miiooai.com` |
| 前端主问题 | 少量模块仍存在直连后端、stub/no-op、旧注释或旧接口口径残留 |
| 后端主问题 | 个别文档与代码漂移，少量生产部署边界需继续收口 |
| 最大阻断点 | 微信登录/绑定链路不一致、API 配置测试返回字段不一致、个别页面仍未完全收口到统一 API 层 |
| 当前执行策略 | 优先修适配层和契约差异，不改 UI 设计和核心交互 |
