# 微信扫码登录回调链路核对（apex 入口缺口与本地不可测）

> 类型：plans / 排障与生产配置核对
> 日期：2026-06-18
> 关联代码：`frontend/src/components/LoginModal.jsx`、`frontend/src/components/WechatOfficialQr.jsx`、`frontend/src/pages/Home.jsx`、`backend/app/routers/auth.py`、`backend/nginx/miiooai-apex.conf`、`backend/scripts/check_wechat_login.sh`

## 1. 现象

用户反馈：微信扫码功能，扫码之后没有登录成功；本地（localhost）测试时表现为「扫描后局部跳转，跳转到登录页面」。

## 2. 根因结论

这不是前端轮询或后端回调的代码 bug，而是 **微信网页扫码登录（`snsapi_login`）的回调域名硬约束 + 本地环境无法登记进微信开放平台** 导致：

1. 二维码由 `WechatOfficialQr.jsx` 以 `self_redirect: true` 渲染，`redirect_uri` 取自 `backend/.env` 的 `WECHAT_OPEN_REDIRECT_URI=https://miiooai.com/`。
2. 本地前端找本地后端 `localhost:8000` 建扫码会话，`state=<本地 session>` 存在本地后端。
3. 扫码确认后，微信把二维码 **iframe** 跳到 `https://miiooai.com/?code=...&state=<本地 session>`，iframe 内加载的是生产站 —— 即用户看到的「局部跳转」。
4. 生产后端拿 `<本地 session>` 查不到（会话在本地后端），且 iframe(`miiooai.com`) 与父窗口(`localhost`) 跨域，`LoginModal.jsx` 的 `event.origin === window.location.origin` 校验丢弃 `postMessage`。
5. 本地轮询持续拿到 `pending` 直至过期，故「扫码后没登录成功」。

localhost 无法登记到微信开放平台，因此 **本地不可能验证真实微信扫码**，只能在生产域名 `https://miiooai.com` 上测。

## 3. 仓内代码链路核对结论（均已齐全自洽）

| 环节 | 状态 | 依据 |
|---|---|---|
| 回调域归一 | 齐全 | `auth.py:217` / `users.py:129` 把旧 `/wx/callback` 自动归一到 `/` |
| 同源 postMessage | 齐全 | 生产下 iframe 跳回 `miiooai.com`，与父窗口同源，`LoginModal.jsx:627` 校验可过 |
| CORS apex→www | 齐全 | `CORS_ORIGINS` 含 `https://miiooai.com`；`cors.py` 自动补 www 变体；`main.py:96` 有正则兜底 |
| 前端不依赖跨站 cookie | 齐全 | `apiPollWechatQrCodeStatus` 直接从 JSON body 取 token 写 localStorage |
| 后端回调接口可达 | 齐全 | `/api/auth/wechat/callback/complete` 已挂载，www 站点 nginx `location /` 全量反代 :8000 |

## 4. 唯一潜在缺口：apex 站点 nginx 缺失

既有 `backend/nginx/miiooai.conf` 的 `server_name` 只有 `www.miiooai.com miiooaib.com`，**没有 apex `miiooai.com`**。微信回调域登记的是 apex，扫码跳回 `https://miiooai.com/?code=...` 时，若 apex 站点未把 `/api` 反代到后端、未把根路径回退到 `index.html`，回调链路会直接 404。

本轮已补：

- `backend/nginx/miiooai-apex.conf` —— apex 站点 server 块模板，含 `location /api/` 反代 :8000 + `location /` 的 `try_files ... /index.html` SPA 回退。
- `backend/scripts/check_wechat_login.sh` —— 线上四段自检（apex 回调入口不 404 / 后端回调接口可达 / CORS apex→www 放行 / 二维码 redirect_uri 主机核对）。

## 5. 需在平台与服务器侧人工确认（代码读不到）

1. 微信开放平台「授权回调域」= `miiooai.com`（仅域名，不带 `https://`、不带路径）。
2. 线上 `backend/.env` 的 `WECHAT_OPEN_REDIRECT_URI=https://miiooai.com/`，与回调域同主机；同步代码后 **重启 `miioo-web`** 才生效。
3. apex 站点与 www 站点必须指向 **同一套后端 + 同一份前端构建产物**，否则两端会话对不上。

## 6. 验证方式

```bash
# 线上自检（服务器执行）
bash backend/scripts/check_wechat_login.sh

# 部署 apex nginx
sudo cp backend/nginx/miiooai-apex.conf /www/server/panel/vhost/nginx/miiooai.com.conf
sudo nginx -t && sudo nginx -s reload
```

四段全 OK 后，在 `https://miiooai.com` 手工扫码确认登录成功。
