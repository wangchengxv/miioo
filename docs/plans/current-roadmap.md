# 当前路线图

## 1. 当前阶段判断

截至 `2026-06-10`，项目已从“P0 主链路补接口 / 补适配 / 去 mock”为主，转入以下新阶段：

- 真实环境联调验收
- 生成任务链路稳定性收口
- 生产环境并发与性能治理
- 文档治理与 Harness 约束收口
- 资产 / 创作 / 剪辑等聚合页体验修整
- 生产组合口径收口，明确 `backend` / `frontend` / `docs` 的职责边界

## 2. 当前优先级

### Priority A：真实环境验收

- 登录、当前用户、通知
- 项目列表 / 新建项目 / 全局设定
- 剧本、主体、分镜主链路
- API 配置真实 key 场景

### Priority B：分镜专项后续阶段

- `SBX-03` 任务2 组合提示词持续收口
- `SBX-04` 分镜页任务1/任务2 展示与编辑承接细化
- `SBX-05` 下游图片/视频/配音统一消费新口径

### Priority C：聚合页闭环验证

- 资产库聚合读取与回收站
- 创作页图片/视频/配音任务链路
- 剪辑成片在线编辑与导出结果回显

### Priority D：文档与约束持续治理

- 持续把新结论写入小文档
- 控制历史母本不再无限膨胀
- 用工具钩子保障核心文档存在
- 固化“生产环境前后端对齐需求”作为联调与上云前的默认需求事实源

### Priority E：生产运行时治理

- 以 `生产环境并发与性能治理计划.md` 为事实源推进共享状态、长任务、轮询、媒体与容量治理
- 优先消除多 worker 下的进程内状态风险和 Web 进程挤占风险
- 在继续做真实环境联调的同时，把性能问题转成代码、配置、脚本与文档约束

## 4. Definition of Done

当前阶段完成至少满足：

- 核心模块真实环境手工联调可按文档复现
- 生成型长任务都有统一任务状态承接
- 文档体系可通过导航快速找到当前事实
- 每次任务都有 changelog 与 progress 回写位置
- 新接手的人可按“需求 -> 可行性 -> 概要设计 -> 详细设计 -> 项目进度 -> 模块 Todo -> 历史归档”的顺序快速进入项目
