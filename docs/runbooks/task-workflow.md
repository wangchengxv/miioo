# 任务执行手册

## 1. 标准闭环

1. 阅读入口文档
2. 锁定页面闭环
3. 对齐前端输入输出
4. 对齐后端契约
5. 设计映射与最小改动路径
6. 实现代码或文档变更
7. 做最小必要验证
8. 回写 mapping、backlog、progress、changelog

## 2. 最小改动优先级

1. 优先改 `frontend/src/api/`
2. 再补后端兼容字段或能力
3. 再补页面最小必要状态承接
4. 最后才考虑改页面命名或交互

## 3. 文档回写模板

- 本次打通了哪个页面或模块
- 改动主要落在哪一层
- 新增或修正了哪些字段映射
- 当前仍遗留什么问题
- 下一个最应该推进的动作是什么

## 4. 异常处理

- 若发现临时调试代码残留，优先清理并记录到 changelog
- 若发现页面和后端事实冲突，先写 mapping 再改代码
- 若只是历史大文件太大，不直接删历史，先建立小文档入口
