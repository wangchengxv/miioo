# 媒体链路验收清单

## 1. 目标

本清单用于承接 `MEDIA-BPLUS-*` 阶段任务完成后的验收动作。

它只回答三件事：

- 当前阶段要验什么
- 需要留什么证据
- 验收结论怎么写

## 2. 适用范围

适用于以下任务包或其后续真实实现阶段：

- `MEDIA-BPLUS-10`
- `MEDIA-BPLUS-20`
- `MEDIA-BPLUS-30`
- `MEDIA-BPLUS-40`
- `MEDIA-BPLUS-50`
- `MEDIA-BPLUS-60`
- `MEDIA-BPLUS-70`

推荐与以下文档配合使用：

- [MEDIA-BPLUS-70-文档回写闭环与验收矩阵实现设计.md](file:///Users/xingyi/Desktop/迭代一版/docs/plans/MEDIA-BPLUS-70-文档回写闭环与验收矩阵实现设计.md)
- [task-workflow.md](file:///Users/xingyi/Desktop/迭代一版/docs/runbooks/task-workflow.md)
- [runtime-result-templates.md](file:///Users/xingyi/Desktop/迭代一版/docs/runbooks/runtime-result-templates.md)

## 3. 使用方式

1. 为本轮任务新建一个验收结果文件，例如：
   - `docs/plans/media-acceptance-<task-id>-YYYY-MM-DD.md`
2. 按本模板填写。
3. 若真实环境联调由你手工验证，则在结果中明确写出“待用户手工验证”的部分。
4. 完成后同步回写：
   - `docs/plans/module-progress.md`
   - `docs/plans/项目进度文档.md`
   - `CHANGELOG.md`

## 5. 必查项速查

- `方案层`：边界没漂移
- `实现层`：功能与字段语义成立
- `运行层`：可排障、可灰度、可回滚
- `文档层`：progress / changelog / API 文档已同步
- `交接层`：下一位接手者能继续推进

## 6. 判定规则

- 若代码或文档已完成，但真实环境尚未联调：
  - 结论写“代码与文档已就绪，真实环境联调待用户手工验证”
- 若本轮不涉及对外契约变化：
  - 明确写“本轮不更新 `BACKEND_API_DOC.md` 与 `接口变动文档.md`”
- 若本轮涉及新字段或行为变化：
  - 验收不应判定为完整通过，直到接口文档同步完成
