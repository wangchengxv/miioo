# 模块边界与约束

## 1. 边界原则

- 前端页面负责产品表达，不负责翻译后端字段
- 前端适配层负责契约翻译，不负责承载复杂页面 UI
- 前端展示链路必须保证字段语义清晰，列表、详情、预览、下载属于同一展示闭环
- 后端 router 负责接口契约，不直接堆砌页面展示语义
- 服务层负责业务逻辑，不把临时调试逻辑长期残留在线上链路
- 文档层负责事实留痕，不重复抄写同一份事实

## 2. 模块边界表

| 模块 | 目录 | 允许依赖 | 禁止事项 |
|---|---|---|---|
| 页面层 | `miioo/frontend/src/pages/` | `components/`、`src/api/`、本地 hooks/lib | 直接写真实请求、直接拼后端字段、直接猜媒体类型或任务结果 |
| 通用组件层 | `miioo/frontend/src/components/` | 基础 UI、受控 props、少量 hooks | 自行请求后端、偷偷维护业务事实 |
| API 适配层 | `miioo/frontend/src/api/` | `http.js`、能力表、少量工具函数 | 把页面布局逻辑写进适配层、遗漏展示字段语义拆分 |
| Router 层 | `miioo/backend/app/routers/` | schemas、services、dependencies | 直接耦合前端局部展示文案、返回模糊媒体语义字段 |
| Service 层 | `miioo/backend/app/services/` | models、外部供应商 SDK、工具模块 | 回写前端特有 UI 状态 |
| 模型层 | `miioo/backend/app/models/` | SQLAlchemy/数据库 | 承担页面拼装职责 |
| Harness 文档层 | 根级 md + `docs/` | 仓库事实、代码路径、阶段结果 | 与代码事实脱节、重复维护整段接口定义 |

|---|---|---|
| `CLAUDE.md` | 原则、边界、主路线 | 逐日变更日志 |

## 4. 跨层调用铁律

- 页面层访问后端，必须通过 `src/api/`
- 页面层只消费视图模型，不负责翻译后端字段、媒体语义或任务状态口径
- 列表、详情、下载、预览必须共享同一适配来源，不得跨层各自拼装
- 媒体展示必须显式区分图片、视频、音频、缩略图、原文件和下载地址
- 生成任务状态，必须通过统一任务状态接口承接
- 历史问题与临时排障，不直接长期堆积到入口文档
- 新增规则优先入仓到小文档，再决定是否回写历史母本

## 5. 前端展示边界

- 页面层负责“怎么展示”，不负责“字段到底代表什么”
- 适配层负责把后端字段转换为页面可直接消费的视图模型，并定义统一回退顺序
- 后端 router 负责保证契约语义明确，避免把视频、缩略图、原文件等不同职责混成同一个模糊字段
- 缓存隔离、任务终态刷新和列表/详情一致性属于展示闭环的一部分，不应被视为纯请求层细节
