# 项目 AGENTS 入口规范

## 1. 身份与权限

- 你的角色是本仓库的执行型 Agent，负责在既有代码和文档事实基础上完成分析、实现、校验与回写。
- 生产目标口径固定为：`backend/` 承接生产后端，`frontend/` 承接生产前端，`docs/` 承接 Harness 文档事实层。
- 可修改目录：`backend/`、`frontend/`、`docs/`、根级规范文档与自动化配置目录。
- 只读优先目录：历史导出 SQL、运行时产物、截图/素材目录、`frontend/src/ref/`。
- 禁止直接删除或覆盖用户未明确授权的历史文档、数据库快照、上传素材和参考代码。

## 2. 标准工作流程

1. 阅读任务相关文档与代码事实源。
2. 确认目标、边界、不做事项和验收标准。
3. 优先在适配层和配置层做最小闭环改动，避免破坏现有 UI/交互。
4. 完成静态校验、必要测试和自查。
5. 回写 `docs/`、进度文档和变更日志，再交付结果。

## 3. 强制约束

- 文档即事实：架构、计划、约束、结论必须入仓，不依赖聊天记忆。
- 机械约束优先：能用代码、脚本、CI、lint、pre-commit 固化的规则，不只写口头约定。
- 前端视觉不随意改动：涉及页面样式时，先读 `frontend/design-system/` 与局部规则文档。
- 页面层不得直接请求后端：所有前端请求统一经 `frontend/src/api/`。
- 前端展示字段必须经适配层统一归一：页面层不得直接猜后端字段语义，也不得在组件内自行拼装兼容字段。
- 列表、详情、预览、下载、导出必须拆分展示语义：禁止混用一个宽泛 `url` 同时承接缩略图、原文件、播放地址和下载地址。
- 涉及媒体展示时，必须先确认当前字段对应图片、视频或音频语义，不得把 `.mp4/.webm/.mov/.mp3` 一类地址直接塞进错误标签。
- 涉及任务结果展示时，必须先确认接口属于同步口径还是任务制口径，禁止把任务创建响应误当最终结果。
- 后端契约以 `backend/BACKEND_API_DOC.md`、`backend/app/routers/` 和已注册路由为准。
- 生产组合需求以 `docs/plans/生产环境前后端对齐需求.md` 为当前任务事实源。
- 每次完成任务都要判断是否更新 `docs/plans/`、`frontend/PROJECT.md`、`CHANGELOG.md`。

## 4. 工具使用规则

- 开始前先读相关规则文档：前端样式读 `frontend/design-system/AGENTS.md`，接口适配读 `frontend/src/api/AGENTS.md`，页面逻辑读 `frontend/src/pages/AGENTS.md`。
- 涉及前端展示新增或修复时，先读 `docs/plans/前端展示约束规范.md`，确认字段语义、任务承接、缓存失效和列表/详情/下载一致性规则。
- 搜索代码优先用结构化检索工具，编辑优先做最小改动，不大面积重写。
- 未经用户明确授权，不做破坏性操作，不回滚他人改动，不删除历史资料。
- 发现代码事实、文档事实或运行口径互相冲突时，先停下来澄清，再继续执行。
- 涉及联调时，以“你手工验证真实环境，我负责代码与文档收口”为默认协作方式。
